# 第 3 阶段分析视图设计

## 目标

构建分析视图与可视化能力，覆盖：

- 文档间关系图
- 查询频率与热门话题分析
- 响应时间与系统资源监控

## 设计原则

- 关系图优先可解释，再叠加向量相似度
- 所有图表使用真实数据
- 不破坏现有问答主界面

## 页面结构

新增“分析视图”入口，进入后展示：

- 顶部概览卡片
- 左侧文档关系图
- 右侧查询分析面板
- 底部系统监控面板

## 关系图

### 节点

- 每个文档一个节点

### 边

- `same_category`
- `same_source`
- `shared_keywords`
- `semantic_similarity`

### 优先级

- 分类、来源、关键词关系作为主解释
- 向量相似度只作为补强信号

### 节点详情

- 文档标题
- 分类
- 来源
- 关联文档数量

## 查询分析

- 查询频率趋势
- 热门话题词
- 热门命中文档
- 检索模式使用分布
- 平均响应延迟

## 系统监控

### 应用层

- 文档数
- 片段数
- 查询总数
- 平均延迟
- 最近错误数（本阶段可先为 0 或缺省）

### 系统层

- CPU 使用率
- 内存使用率
- 磁盘占用率

## 后端接口

新增分析接口：

- `GET /analytics/overview`
- `GET /analytics/document-graph`
- `GET /analytics/query-insights`
- `GET /analytics/system-metrics`

## 技术路线

- 关系图布局先使用轻量 SVG 自绘
- 趋势图与条形图先使用前端原生 SVG/HTML 渲染
- 系统资源采集使用标准库或轻量依赖

## 非目标

- 真正的知识图谱语义推理
- 实时 WebSocket 推送
- 高级告警系统
