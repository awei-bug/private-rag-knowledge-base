# 第 2 阶段设置中心设计

## 目标

将当前右上角“设置”从说明性弹层升级为真正可用的设置中心，并让设置实际驱动系统行为。

## 本阶段范围

- 检索参数配置
- 模型模式展示与切换
- 默认同步目录配置
- 可持久化的用户设置

## 用户可配置项

### 检索

- 默认检索模式：`precise` / `semantic` / `hybrid`
- `top_k`

### 同步

- 默认同步目录

### 模型

- 运行模式标签展示
- Embedding provider 展示
- LLM provider 展示

说明：本阶段允许用户在前端选择“偏本地”或“偏 API”的运行模式，但真实 provider 切换只在后端可用配置合法时生效。

## 数据存储

新增本地配置表，用于持久化设置：

- `default_folder_path`
- `default_retrieval_mode`
- `top_k`
- `preferred_runtime_mode`

## 后端接口

### 读取设置

- `GET /config`
  - 返回系统只读配置
  - 返回用户设置

### 更新设置

- `PUT /config/preferences`
  - 更新用户设置

## 应用行为

- 前端启动时加载设置
- 检索模式默认值由设置初始化
- 提问时使用设置中的默认模式或当前会话切换值
- `top_k` 更新后影响后端检索返回规模
- 同步目录输入框默认取设置值

## UI 设计

设置弹层拆为三组卡片：

- 检索设置
- 运行模式
- 知识库目录

所有可编辑项必须有保存按钮或立即生效反馈。

## 非目标

- 完整高级参数面板
- 动态切换所有 `.env` 级配置
- 多用户隔离设置
